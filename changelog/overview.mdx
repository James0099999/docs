<Update label="May 22, 2026" description="v1.0">

## 🎉 iGamingAce API is live

The first public release of the iGamingAce API is now available. Integrate slots and live games through a single platform, with full control over player wallets.

### New features

- **Main API** — Discover providers, list games, launch sessions, and manage bonus rounds. See the [Main API overview](/mainapi/overview).
- **Seamless Wallet API** — Keep your backend as the source of truth. iGamingAce sends real-time callbacks for [`balance`](/seamless/balance), [`bet`](/seamless/bet), [`win`](/seamless/win), [`betwin`](/seamless/betwin), and [`cancel`](/seamless/cancel) so player funds stay in your system. Read the [Seamless Wallet overview](/seamless/overview).
- **Bonus call workflow** — Register and cancel free-round and free-bet campaigns programmatically with the new [bonus call endpoints](/gettingstarted/bonuscall).
- **Multi-language support** — Launch games in any supported locale using the new language codes reference. See [Supported languages](/gettingstarted/languages).
- **Sandbox environment** — Browse and try live demo slots before going live at [sandbox.igamingace.com](https://sandbox.igamingace.com).

### Updates

- **Refined seamless callback contracts** — Parameter types and descriptions across all wallet callbacks have been clarified so you can implement against an unambiguous spec. Notable items: `transactionId` and `roundId` are `long`, `bonusId` is `long` (use `0` for no bonus), `isBonusBusy` is a boolean, and `currency` follows ISO-4217.
- **Idempotency guidance** — Every transactional callback now documents how to use `transactionId` to safely deduplicate retries.
- **3-second response window** — Timeout and retry behavior is now spelled out for every callback. If your wallet doesn't respond within 3 seconds, iGamingAce sends a `cancel` to revert the round instead of retrying.
- **Standardized error codes** — A single [error codes reference](/gettingstarted/error-codes) covers status values returned across the platform.

</Update>
